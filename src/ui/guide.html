Integration Guide

1. Getting started

The engine is dependency-free. Where you copy the engine folder depends on how your page loads:

// static page in public/ (libs via importmap, like this repo's scenes): src/sounds/ → public/sounds/ · then import Analyzer from '/sounds/Analyzer.js' // bundled entry (libs from npm, e.g. src/main.js importing 'three'): src/sounds/ → your src/sounds/ · then import Analyzer from './sounds/Analyzer.js' public/tracks/ → your public/tracks/ · music (or mic), both cases

Register vite-plugin-vj-tracks.js too. It keeps tracks.json in sync, so dropping an .mp3 in public/tracks/ adds it to the playlist:

import vjTracksPlugin from './vite-plugin-vj-tracks.js' export default defineConfig( { plugins: [ vjTracksPlugin() ] } )

On a brand-new project the first pnpm dev falls back to the mic, because tracks.json is written just after the initial scan. Restart once, or commit a generated tracks.json.

Then read the signals each frame:

const audio = new Analyzer() audio.onAudio( a => { a.volume // instant loudness 0..1 a.volumeSmooth // loudness, smoothed 0..1 a.kick // spikes on each kick 0..1 a.kickHard // strong beats only 0..1 a.volumeByFrequency // per-frequency loudness Float32Array(256), 0..1 } )

Run pnpm dev and open your page. It plays the tracks (or mic), so your visuals react on their own.

Press d for a live debug overlay of every signal (spectrum + meters). Drop more .mp3s in public/tracks/, switch with . / ,, and toggle the mic with m.

2. The helper modules

The Analyzer lazy-loads these in standalone mode, so reach for them only when you want manual control. All three live in src/sounds/.

// AnalyzerDebug- live overlay import AnalyzerDebug from '/sounds/AnalyzerDebug.js' const debug = new AnalyzerDebug( audio ) debug.toggle() debug.show() debug.hide() debug.dispose() // SoundPlayer- music, at audio.player audio.player?.useTrack( url ) audio.player?.nextTrack() audio.player?.prevTrack() audio.player?.useMic() // PlayerControl- now playing widget import PlayerControl from '/sounds/PlayerControl.js' new PlayerControl( { getAudioEl, getSource, getTrackName, onSkip, onSeek } )

3. Integrate it here

Add your scene as a local folder and open a pull request (no list to edit). Copy the template, rename it, and edit its index.html:

public/vj-scenes/template/ → public/vj-scenes/your-name/ # edit index.html, save, then open a PR

Save and the dev server auto-detects the folder. Once merged it joins the loop: the host loads it in an iframe and the same Analyzer switches to receive mode, reading the show's live audio instead of its own. Nothing in your code changes.
